## 함수란?

함수는 자바스크립트에서 가장 중요한 핵심 개념

> 스코프, 실행 컨텍스트, 클로저, 생성자 함수에 의한 객체 생성, 메서드, this, 프로토타입, 모듈화 등이 함수와 깊은 관련이 있다. 함수는 재료를 투입받아 제품을 생산하는 기계와 같다.

**함수는 일련의 과정을 문으로 구현하고 코드 블록으로 감싸 하나의 실행 단위로 정의한 것**

```js
function add(x, y) {
  return x + y;
}

add(2, 5); // 7
```

- 매개 변수 - 입력을 전달 받는 변수
- 인수 - 입력
- 반환값 - 출력

## 함수를 사용하는 이유

중복된 코드를 줄이고, 같은 로직을 여러 곳에서 재사용할 수 있다. 버그가 생겼을 때도 한 군데만 고치면 되니 유지보수가 편하다. 이름을 잘 지으면 주석 없이도 코드가 읽힌다.

## 함수 리터럴

함수도 함수 리터럴로 생성할 수 있다.

```js
let f = function add(x, y) {
  return x + y;
}
```

함수 리터럴도 평가되면 값이 되는데, 그 값이 객체다. 일반 객체와 다른 점은 호출할 수 있다는 것.

## 함수 정의

함수를 정의하는 방법 4가지

```js
// 함수 선언문
function add(x, y) {
  return x + y;
}

// 함수 표현식
let add = function (x, y) {
  return x + y;
}

// Function 생성자 함수
let add = new Function('x', 'y', 'return x + y');

// 화살표 함수(ES6)
let add = (x, y) => x + y;
```

### 함수 선언문

함수 선언문은 함수 이름을 생략할 수 없고, 표현식이 아닌 문이다.

```js
// 기명 함수 리터럴을 단독으로 사용하면 함수 선언문으로 해석
function foo() { console.log('foo'); }
foo(); // foo

// 피연산자로 사용하면 함수 리터럴 표현식으로 해석
(function bar() { console.log('bar'); });
bar(); // ReferenceError: bar is not defined
```

`bar`가 `ReferenceError`인 이유는, 함수 리터럴 표현식으로 해석되면 이름은 함수 몸체 내부에서만 유효하기 때문이다. 반면 함수 선언문은 JS 엔진이 함수 이름과 동일한 식별자를 암묵적으로 만들어 함수 객체를 할당해준다. 그래서 결국 **함수는 함수 이름이 아니라 식별자로 호출하는 것**이다.

### 함수 표현식

자바스크립트 함수는 일급 객체다. 값처럼 변수에 할당하거나 인수로 전달하거나 반환값으로 쓸 수 있다.

```js
let add = function (x, y) {
  return x + y;
}

console.log(add(2, 5)); // 7
```

함수 표현식에서는 함수 이름을 생략하는 게 일반적이다. 기명으로 써도 이름은 함수 몸체 내부에서만 유효하다.

```js
let add = function foo(x, y) {
  return x + y;
}

console.log(add(2, 5)); // 7
console.log(foo(2, 5)); // ReferenceError: foo is not defined
```

### 함수 생성 시점과 호이스팅

```js
console.dir(add); // f add(x, y)
console.dir(sub); // undefined

console.log(add(2, 5)); // 7
console.log(sub(2, 5)); // TypeError: sub is not a function

function add(x, y) {
  return x + y;
}

let sub = function (x, y) {
  return x - y;
};
```

함수 선언문은 런타임 이전에 함수 객체 자체가 만들어진다. 그래서 선언 전에 호출해도 된다. 반면 함수 표현식은 변수 선언만 먼저 되고(`undefined`로 초기화), 실제 함수 객체는 해당 라인이 실행될 때 할당된다. 이게 함수 호이스팅과 변수 호이스팅의 차이다.

> [!TIP]
> React에서 이벤트 핸들러를 함수 표현식(`const handleClick = () => {}`)으로
> 쓰는 이유도 여기 있다. 컴포넌트 내부에서 선언 순서에 덜 의존하게 되고,
> 의도치 않은 호이스팅 동작을 피할 수 있다.

### Function 생성자 함수

> [!WARNING]
> `new Function()`으로 함수를 만드는 방식은 클로저를 생성하지 않는 등 일반적인 함수와 다르게 동작한다. 
> 쓸 일이 거의 없고, 쓰면 안 되는 쪽에 가깝다.

### 화살표 함수

ES6에서 도입됐다. 표현만 짧은 게 아니라 내부 동작도 다르다. `this` 바인딩 방식이 다르고, `prototype` 프로퍼티가 없으며, `arguments` 객체도 생성하지 않는다.

```js
const add = (x, y) => x + y;
console.log(add(2, 5)); // 7
```

## 함수 호출

### 매개변수와 인수

함수가 호출되면 매개변수가 암묵적으로 생성되고 `undefined`로 초기화된 뒤 인수가 순서대로 할당된다. 인수가 부족하면 나머지는 `undefined`로 남고, 초과하면 조용히 무시된다. JS가 개수를 체크하지 않기 때문에 생각보다 자주 실수가 생기는 지점이다.

### 매개변수의 최대 개수

딱히 제한은 없지만, 많을수록 함수가 너무 많은 일을 하고 있다는 신호다. 이상적인 함수는 하나의 일만 한다.

## 참조에 의한 전달과 외부 상태의 변경

```js
function changeVal(primitive, obj) {
  primitive += 100;
  obj.name = 'Kim';
}

let num = 100;
let person = { name: 'Lee' };

changeVal(num, person);

console.log(num);    // 100 - 원시값은 복사되어 전달되므로 원본 유지
console.log(person); // { name: 'Kim' } - 참조값이 전달되므로 원본 변경됨
```

> [!WARNING]
> 객체를 함수에 넘기면 참조값이 복사되어 전달되기 때문에, 함수 내부에서 변경하면 외부 원본도 바뀐다.
> 의도하지 않은 부수 효과가 생기기 쉬운 패턴이다.
> 방어하려면 깊은 복사 후 새 객체를 반환하는 방식으로 설계해야 한다.

## 다양한 함수의 형태

### 즉시 실행 함수

```js
(function () {
  let a = 3;
  let b = 5;
  return a * b;
}());
```

정의하자마자 실행되고 다시 호출할 수 없다. 변수 충돌을 막고 스코프를 격리할 때 쓰던 패턴이다. 

### 재귀 함수

```js
let factorial = function foo(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1);
}

console.log(factorial(5)); // 120
```

자기 자신을 호출하는 함수. 탈출 조건이 없으면 무한 루프에 빠지므로 반드시 종료 조건을 넣어야 한다. 대부분 `for`나 `while`로 대체 가능하다.

### 중첩 함수

```js
function outer() {
  let x = 1;

  function inner() {
    let y = 2;
    console.log(x + y); // 3
  }

  inner();
}

outer();
```

내부 함수는 외부 함수의 변수를 참조할 수 있다. 외부 함수를 돕는 헬퍼 역할이 목적이다.

### 콜백 함수

함수를 인수로 받는 함수를 고차 함수, 그 인수로 전달되는 함수를 콜백 함수라고 한다. 고차 함수가 콜백의 호출 시점을 결정한다.

```js
function repeat(n, f) {
  for (let i = 0; i < n; i++) {
    f(i);
  }
}

repeat(5, (i) => console.log(i));       // 0, 1, 2, 3, 4
repeat(5, (i) => { if (i % 2) console.log(i); }); // 1, 3
```

`map`, `filter`, `reduce`가 대표적인 고차 함수다. 비동기 처리에서도 핵심 패턴으로 쓰인다.

```js
const res = [1, 2, 3].map((item) => item * 2);    // [2, 4, 6]
const odd = [1, 2, 3].filter((item) => item % 2); // [1, 3]
const sum = [1, 2, 3].reduce((acc, cur) => acc + cur, 0); // 6
```

### 순수 함수와 비순수 함수

```js
// 순수 함수 - 같은 입력이면 항상 같은 출력, 외부 상태 건드리지 않음
function increase(n) {
  return ++n;
}

// 비순수 함수 - 외부 변수를 직접 바꿈
let count = 0;
function increase() {
  return ++count; // 호출할 때마다 외부 상태가 달라짐
}
```

순수 함수는 테스트 짜기도 쉽고 디버깅도 편하다. 어디서 호출하든 동일하게 동작하기 때문이다. 비순수 함수는 외부 상태를 바꾸기 때문에 어디서 무엇이 변경됐는지 추적이 어려워진다. 함수형 프로그래밍은 이 부수 효과를 최소화하는 방향으로 설계하는 패러다임이다.
