
## 변수의 생명 주기
### 지역 변수의 생명 주기

>[!IMPORTANT]
>지역 변수의 생명 주기
>지역 변수의 생명 주기는 함수의 생명 주기와 일치한다. 함수가 호출 될 때 생성 되고, 종료될 때 소멸한다.

- 함수 내부에서 선언한 변수는 함수 호출 직후, 다른 코드가 실행되기 이전에 먼저 선언되고 `undefined`로 초기화됨
- 클로저처럼 누군가 스코프를 참조하고 있으면. 지역 변수는 함수가 종료된 후에도 가비지 콜렉터가 해제할 때까지 생존할 수 있음

```js
var x = 'global';

function foo() {
  console.log(x); // undefined (지역 변수 x가 호이스팅되어 초기화 됨)
  var x = 'local';
}

foo();
console.log(x); // global
```

>[!IMPORTANT]
>호이스팅은 스코프 단위로 동작
>전역 변수 호이스팅은 전역 전체에, 지역 변수 호이스팅은 해당 함수 스코프 내에서만 유효하다.

---
### 전역 변수의 생명주기

>[!IMPORTANT]
>전역 객체(Global Object)
>코드 실행 이전에 가장 먼저 생성되는 특수한 객체.
>브라우저에서는 `window`, Node.js에서는 `global`

- `var`로 선언한 전역 변수는 전역 객체의 프로퍼티가 됨 -> 전역 변수의 생명주기 = 전역 객체의 생명 주기
- 브라우저 환경에서는 웹페이지를 닫을 때까지 유효

---

## 전역 변수의 문제점

>[!IMPORTANT]
>암묵적 결합(implicit coupling)
>모든 코드가 전역 변수를 참조하고 변경할 수 있는 상태.
>유효 범위가 클수록 의도치 않은 상태 변경 위험이 높아진다.

>[!IMPORTANT]
>긴 생명 주기
>전역 변수는 앱이 종료될 때까지 메모리를 점유.
>`var`는 중복 선언이 허용되므로 의도치 않은 재할당이 발생할 수 있음.

>[!IMPORTANT]
>스코프 체인 종점
>전역 변수는 스코프 체인 탐색 시 가장 마지막에 검색됨 -> 검색 속도가 가장 느림.

>[!IMPORTANT]
>네임스페이스 오염
>파일이 분리되어 있어도 JS는 하나의 전역 스코프를 공유.
>다른 파일에서 같은 이름의 전역 변수/함수가 존재하면 충돌 발생.

## 전역 변수의 사용을 억제하는 방법

>전역 변수를 반드시 사용해야 할 이유를 찾지 못한다면 지역 변수를 사용.
>변수의 스코프는 좁을수록 좋다.

### 즉시 실행 함수(IIFE)

모든 코드를 IIFE로 감싸면 모든 변수가 지역 변수가 된다. 라이브러리 등에서 자주 사용하는 패턴.

```js
(function () {
  var foo = 100; // 지역 변수
}());

console.log(foo); // ReferenceError: foo is not defined
```

---

### 네임스페이스 객체

- 전역 객체를 하나 만들고 변수를 프로퍼티로 추가하는 방식
- 식별자 충돌은 방지할 수 있으나, 네임스페이스 객체 자체가 전역 변수이므로 근본적인 해결책은 아님

---

### 모듈 패턴

>[!IMPORTANT]
>모듈 패턴 / 캡슐화 / 정보 은닉
>클로저를 이용해 IIFE로 변수와 함수를 묶어 모듈화.
>전역 변수 억제와 함께 `private`변수 구현(정보 은닉)이 가능하다.

```js
var Counter = (function () {
  var num = 0; // private 변수
  
  return {
    increase() { return ++num; },
    decrease() { return --num; }
  };
}());

console.log(Counter.num); // undefined (외부 접근 불가)
console.log(Counter.increase()); // 1
console.log(Counter.decrease()); // 0
```

### ES6 모듈

>[!IMPORTANT]
>ES6 모듈/ 모듈 스코프
>ES6 모듈은 파일 자체의 독자적인 모듈 스코프를 제공.
>모듈 내 `var`선언 변수는 전역 변수도, `window`프로퍼티도 아니다.

```html
<script type="module" src="lib.mjs"></script>
```
