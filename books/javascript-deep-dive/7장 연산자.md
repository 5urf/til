## == vs ===

`==`(동등 연산자)는 비교 전에 암묵적 타입 변환을 수행한다.

```js
"1" == 1; // true
0 == false; // true
null == undefined; // true
```

`==` 타입 변환 순서:

1. 타입이 같으면 `===`처럼 비교
2. `null == undefined` - true (특별 규칙)
3. 한쪽이 숫자, 한쪽이 문자열 - 문자열을 숫자로 변환
4. 한쪽이 불리언 - 불리언을 숫자로 변환 (`true` → 1, `false` → 0)
5. 한쪽이 객체, 한쪽이 원시값 - 객체를 원시값으로 변환 (`valueOf` → `toString` 순)

변환 규칙 자체가 예측하기 어렵기 때문에 항상 `===`를 써야 한다.

`===`(일치 연산자)는 타입 변환 없이 타입과 값 모두 같아야 true다.

```js
NaN === NaN; // false - NaN은 자신과도 일치하지 않는 유일한 값
Number.isNaN(NaN) + // true - NaN 여부 확인은 반드시 Number.isNaN() 사용
  0 ===
  -0; // true
Object.is(+0, -0); // false - 구분이 필요한 경우 Object.is 사용
```

## typeof null은 "object" - JS 초기 설계의 버그

`typeof` 연산자는 피연산자의 타입을 문자열로 반환한다. 반환값은 `"string"`, `"number"`, `"boolean"`, `"undefined"`, `"symbol"`, `"object"`, `"function"` 7가지.

```js
typeof null; // "object" - null은 원시값인데 "object"를 반환 - 버그
```

하위 호환성 때문에 수정되지 않고 지금까지 유지되고 있다. null 체크를 `typeof`로 하면 안 되는 이유가 바로 이것이다.

```js
typeof null === "object"; // true - 잘못된 체크
null === null; // true - 이렇게 체크해야 한다
```

> [!TIP]
> 선언하지 않은 변수에 `typeof`를 사용해도 에러가 나지 않고 `"undefined"`를 반환한다.

```js
typeof undeclaredVar; // "undefined" - 에러 없음
```

## 전위 ++ vs 후위 ++

후위(`x++`)는 현재 값을 먼저 반환한 뒤 증가. 전위(`++x`)는 먼저 증가한 뒤 그 값을 반환.

```js
let x = 5;
console.log(x++); // 5 - 반환 후 증가
console.log(x); // 6

let y = 5;
console.log(++y); // 6 - 증가 후 반환
console.log(y); // 6
```

단순히 증가만 시킬 때는 차이가 없지만, 반환값을 바로 사용하는 경우 결과가 달라진다.

```js
let a = 0;
let b = a++; // b = 0, a = 1
let c = ++a; // c = 2, a = 2
```
