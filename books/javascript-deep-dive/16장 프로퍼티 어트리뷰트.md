## 내부 슬롯과 내부 메서드
>ECMAScript 사양에서 `[[ ]]`로 감싼 이름들이 내부 슬롯과 내부 메서드다.
>자바스크립트 엔진의 내부 로직이므로 원칙적으로 직접 접근할 수 없고, 일부에 한해 간접 수단을 제공

- 예: `[[Prototype]]` 내부 슬롯 -> `__proto__`로 간접 접근 가능

## 프로퍼티 어트리뷰트

>[!IMPORTANT]
>프로퍼티 어트리뷰트 / 프로퍼티 디스크립터
>자바스크립트 엔진은 프로퍼티를 생성할 때 프로퍼티의 상태를 나타내는 프로퍼티 어트리뷰트를 기본 값으로 자동 정의한다.
>프로퍼티 어트리뷰트는 내부 슬롯 `[[Value]]`,`[[Writable]]`,`[[Enumerable]]`,`[[Configurable]]` 이다.

- 직접 접근 불가. `Object.getOwnPropertyDescriptor`로 간접 확인 가능
- 반환값이 **프로퍼티 디스크립터 객체**

```js
const person = { name: 'Lee' };
console.log(Object.getOwnPropertyDescriptor(person, 'name'));
// {value: "Lee", writable: true, enumerable: true, configurable: true}

// ES8: 모든 프로퍼티의 디스크립터 객체들 반환
console.log(Object.getOwnPropertyDescriptors(person));
```

## 데이터 프로퍼티와 접근자 프로퍼티

>[!IMPORTANT]
>데이터 프로퍼티 vs 접근자 프로퍼티
>데이터 프로퍼티는 키와 값으로 구성된 일반적인 프로퍼티.
>접근자 프로퍼티는 자체적으로 값을 갖지 않고 getter/setter 함수로 구성된 프로퍼티다.

- 데이터 프로퍼티 어트리뷰트: `[[Value]]`, `[[Writable]]`, `[[Enumerable]]`,`[[Configurable]]`
- 접근자 프로퍼티 어트리뷰트: `[[Get]]`, `[[Set]]`, `[[Enumerable]]`, `[[Configurable]]`

```js
const person = {
  firstName: 'Yun',
  lastName: 'Lee',
  get fullName() { // getter
    return `${this.firstName} ${this.lastName}`;
  },
  set fullName(name) { // setter
    [this.firstName, this.lastName] = name.split(' ');
  }
};

person.fullName = 'Hak Lee'; // setter 호출
console.log(person.fullName); // Hak Lee - getter 호출
```

>[!TIP]
>접근자 프로퍼티는 `[[Value]]`를 가지지 않는다.
>디스크립터로 확인하면 `{get: f, set: f, enumerable: true, configurable: true}`
>형태로 나온다.

## 객체 변경 방지

>[!IMPORTANT]
>Object.freeze / Object.seal / Object.preventExtensions
>객체의 변경을 방지하는 3단계 메서드.
>금지하는 강도가 다르다.

| 구분   | 메서드                 | 추가  | 삭제  | 읽기  | 쓰기  | 어트리뷰트 재정의 |
| ---- | ------------------- | --- | --- | --- | --- | --------- |
| 확장금지 | `preventExtensions` | x   | o   | o   | o   | o         |
| 밀봉   | `seal`              | x   | x   | o   | o   | x         |
| 동결   | `freeze`            | x   | x   | o   | x   | x         |

- 확인 메서드: `Object.isExtensible()`, `Object.isSealed()`,`Object.isFrozen()`

>[!WARNING]
>위 메서드들은 모두 **얕은 변경 방지**(shallow only)다.
>직속 프로퍼티만 변경이 방지되고 중첩 객체까지는 영향을 주지 못한다.
>중첩 객체까지 동결하려면 재귀적으로 `Object.freeze`를 호출해야 한다.

```js
function deepFreeze(target) {
  if(target && typeof target === 'object' && !Object.isFrozen(target)) {
    Object.freeze(target);
    Object.keys(target).forEach(key => deepFreeze(target[key]));
  }
  return target;
}
```
