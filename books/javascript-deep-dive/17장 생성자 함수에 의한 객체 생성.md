## 생성자 함수

>[!IMPORTANT]
>`new`연산자와 함께 호출해 객체(인스턴스)를 생성하는 함수.
>
>생성자 함수로 만들어진 객체를 **인스턴스(instance)**라 한다.

- JS는 Object 외에도 `String`,`Number`,`Boolean`,`Function`,`Array`,`Date`,`RegExp`,`Promise`등 빌트인 생성자 함수를 제공한다.

>[!NOTE]
>`new Object()`로 빈 객체를 만든 뒤 프로퍼티를 추가하는 방식은 가능하지만,
>
>특별한 이유가 없다면 객체 리터럴이 더 간편하다.

---

## 객체 리터럴의 한계

>객체 리터럴은 하나의 객체만 생성한다.
>
>동일한 프로퍼티 구조의 객체가 여러 개 필요하면 같은 내용을 반복해야해 비효율적이다.

```js
// 구조가 같아도 매번 새로 기술해야 한다
const circle1 = { radius: 5, getDiameter() { return 2 * this.radius; } };
const circle2 = { radius: 10, getDiameter() { return 2 * this.radius; } };
```

- 생성자 함수는 이 문제를 해결한다.
	- 템플릿(클래스)처럼 동작해 동일한 구조의 객체를 여러개 간편하게 생성할 수 있다.

```js
function Circle(radius) {
  this.radius = radius;
  this.getDiameter = function () {
    return 2 * this.radius;
  };
}

const circle1 = new Circle(5);
const circle2 = new Circle(10);
```

## this 바인딩

>[!IMPORTANT]
>`this`가 가리키는 값은 **함수 호출 방식에 따라 동적으로 결정**된다.

| 함수 호출 방식   | this가 가리키는 값             |
| ---------- | ------------------------ |
| 일반 함수로 호출  | 전역 객체(`window`/`global`) |
| 메서드로 호출    | 마침표 앞의 객체                |
| 생성자 함수로 호출 | 생성자 함수가 (미래에) 생성할 인스턴스   |

>[!WARNING]
>`new`없이 생성자 함수를 호출하면 일반 함수로 동작한다.
>
>반환문이 없으므로 `undefined`를 반환하고,
>
>`this`는 전역 객체를 가리켜 의도치 않게 전역 프로퍼티가 생긴다.

## 인스턴스 생성 3단계

>[!IMPORTANT]
>`new`로 생성자 함수를 호출하면 JS엔진이 다음 3단계를 암묵적으로 처리한다.

1. **인스턴스 생성과 this 바인딩**
	- 암묵적으로 빈 객체(인스턴스)가 생성되고 `this`에 바인딩된다.
	- 함수 몸체 코드가 실행되기 전, 런타임 이전에 처리된다.
2. **인스턴스 초기화**
	- `this`에 바인딩된 인스턴스에 프로퍼티 메서드를 추가하고 초기값을 할당한다.
	- 이 처리는 개발자가 직접 기술한다.
3. **인스턴스 반환**
	- 모든 처리가 끝나면 완성된 인스턴스가 바인딩된 `this`가 암묵적으로 반환된다.

>[!WARNING]
>생성자 함수 내부에서 **다른 객체**를 명시적으로 `return`하면 `this`가 아닌 그 객체가 반환된다.
>
>**원시값**을 명시적으로 `return`하면 무시되고 `this`가 반환된다.
>
>생성자 함수 내부에서는 `return`문을 생략해야 한다.

>[!NOTE]
>**바인딩**: 식별자와 값을 연결하는 과정.
>
>`this`바인딩은 `this`키워드와 `this`가 가리킬 객체를 연결하는 것이다.

## `[[Call]]`과 `[[Construct]]`

>[!IMPORTANT]
>함수는 객체이므로 일반 객체의 내부 슬롯/메서드를 모두 가진다.
>
>여기에 더해 `[[Call]]`과 `[[Construct]]` 내부 메서드를 추가로 가진다.
>
>- 일반 함수로 호출 -> `[[Call]]`호출
>
>- `new`와 함께 호출 -> `[[Construct]]`호출

| 용어              | 의미                                          |
| --------------- | ------------------------------------------- |
| callable        | `[[Call]]`을 갖는 함수. **모든**함수 객체는 callable 이다 |
| constructor     | `[[Construct]]`를 갖는 함수. 생성자 함수로 호출 가능       |
| non-constructor | `[[Construct]]`를 갖지 않는 함수. `new`로 호출하면 에러   |

>[!WARNING]
>모든 함수는 callable이지만, 모든 함수가 constructor인 것은 아니다.

## constructor와 non-constructor 구분

>[!IMPORTANT]
>JS엔진은 **함수 정의 방식**을 기준으로 구분한다. 어디에 할당됐는지가 아니다.
>- constructor: 함수 선언문, 함수 표현식, 클래스
>- non-constructor: ES6 메서드 축약 표현(`x() {}`), 화살표 함수 (`() => {}`)

```js
const arrow = () => {};
new arrow(); // TypeError: arrow is not a constructor

const obj = { x() {} }; // ES6 메서드 축약 표현
new obj.x(); // TypeError: obj.x is not a constructor
```

>[!WARNING]
>ECMAScript 사양에서 **메서드**란 ES6 메서드 축약 표현만을 의미한다.
>
>객체 프로퍼티에 일반 함수 표현식으로 할당한 것은 메서드로 인정하지 않으며, constructor로 취급된다.

## new.target

>[!IMPORTANT]
>ES6에서 도입된 메타 프로퍼티.
>
>constructor인 모든 함수 내부에서 암묵적인 지역 변수 처럼 사용된다.
>- `new`와 함께 생성자 함수로 호출 -> `new.target` = 함수 자신
>- 일반 함수로 호출 -> `new.target` = `undefined`

- 함수 내부에서 `new.target`을 확인해 `new`없이 호출됐을 경우 재귀 호출로 생성자 함수로 강제할 수 있다

```js
function Circle(radius) {
  if (!new.target) {
    return new Circle(radius);
  }
  this.radius = radius;
  this.getDiameter = function () {
    return 2 * this.radius;
  };
}

const circle = Circle(5); // new 없이 호출해도 정상 동작
```

>[!TIP]
>`new.target`을 지원하지 않는 환경(IE)에서는 **스코프 세이프 생성자 패턴**을 사용한다.
>
>`if (!(this instanceof Circle))`조건으로 동일한 방어 로직을 구현할 수 있다.

>[!NOTE]
>`Object`,`Function`빌트인 생성자 함수는 `new`없이 호출해도 동일하게 동작한다.
>반면 `String`, `Number`, `Boolean`,은 `new`없이 호출하면 원시값을 반환한다. 타입 변환에 활용된다.
