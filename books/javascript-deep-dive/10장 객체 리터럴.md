## 객체란?

> JS는 객체 기반의 프로그래밍 언어.
> JS를 구성하는 거의 모든 것이 객체.
> **원시 값을 제외한 나머지 값(변경 가능한 값)은 모두 객체**
> 프로퍼티와 메서드로 구성된 복합 자료구조

- 프로퍼티 : 객체의 상태를 나타내는 값
- 메서드 : 프로퍼티를 참조하고 조작할 수 있는 동작

## 객체 리터럴에 의한 객체 생성

- 객체 리터럴 (객체를 생성하기 위한 표기법) `{}`
- Object 생성자 함수
- 생성자 함수
- `Object.create` 메서드
- 클래스(ES6)

```js
const person = {
 name: 'Lee',
 helloWorld: function (){
   console.log(`Hello! ${this.name}`);
 }
};

console.log(typeof person); // object
console.log(person); // {name: "Lee", helloWorld: f}
```

>[!WARNING]
>중괄호는 코드 블록이 아니라 표현식이므로 세미콜론`;`을 붙인다.

## 프로퍼티

>객체는 프로퍼티의 집합이며, 프로퍼티는 키와 값으로 구성된다.

```js
const person = {
  // 프로퍼티 키는 name, 프로퍼티 값은 'Lee'
  name: 'Lee'
};
```

프로퍼티 키와 프로퍼티 값으로 사용할 수 있는 값

- 프로퍼티 키 : 빈 문자열을 포함하는 모든 문자열 또는 심벌 값
- 프로퍼티 값 : JS에서 사용할 수 있는 모든 값

```js
const person = {
  firstName: 'YunHak',
  last-name: 'Lee' // SyntaxError: Unexpected token -
};
```

```js
const person = {
  name: 'Kim',
  name: 'Lee'
};

console.log(person); // {name: "Lee"}
```

>[!WARNING]
>식별자 네이밍 규칙을 따르지 않는 이름에는 반드시 따옴표를 사용해야 한다.
>프로퍼티 키를 중복 선언하면 나중에 선언한 프로퍼티 값이 덮어쓴다.


## 메서드

>프로퍼티 값이 함수일 경우 일반 함수와 구분하기 위해 메서드라 부름
>메서드는 객체에 묶여 있는 함수를 의미

```js
const circle = {
  radius: 5, // 프로퍼티
  
  // 메서드
  getDiameter: function () { 
    return 2 * this.radius; // this는 circle를 가리킴
  }
};

console.log(circle.getDiameter()); // 10
```

## 프로퍼티 접근

프로퍼티에 접근하는 방법 두가지

- 마침표 표기법 `.` (마침표 프로퍼티 접근 연산자)
- 대괄호 표기법 `[ ... ]` (대괄호 프로퍼티 접근 연산자)

```js
const person = {
 name: 'Lee',
};

console.log(person.name); // Lee
console.log(person['name']); // Lee
```

```js
const person = {
 name: 'Lee',
};

console.log(person[name]); // ReferenceError
console.log(person.age); // undefined
```

>[!WARNING]
>대괄호 프로퍼티 접근 연산자 내부에 지정하는 프로퍼티 키는 반드시 따옴표로 감싼 문자열이어야 한다.
>객체에 존재하지 않는 프로퍼티에 접근하면 `undefined`를 반환

## 프로퍼티 값 갱신, 동적 생성, 삭제

이미 존재하는 프로퍼티에 값을 할당하면 프로퍼티 값 갱신

```js
let person = {
 name: 'Lee',
};

person.name = 'Kim'

console.log(person); // {name: "Kim"}
```

존재하지 않는 프로퍼티에 값을 할당하면 프로퍼티가 동적 생성, 추가 프로퍼티 값 할당

```js
let person = {
 name: 'Lee',
};

person.age = 20

console.log(person); // {name: "Lee", age: 20}
```

>`delete` 연산자는 객체의 프로퍼티를 삭제하며 표현식이어야 한다.
>존재하지 않는 프로퍼티를 삭제하면 에러 없이 무시

```js
let person = {
 name: 'Lee',
};

person.age = 20

delete person.age;

delete person.address;

console.log(person); // {name: "Lee"}
```

## ES6 객체 리터럴의 확장 기능

```js
let x = 1;
let y = 2;

const obj = { x, y };

console.log(obj); // {x: 1, y: 2}
```

### 계산된 프로퍼티 이름

```js
const prefix = 'prop';
let i = 0;

const obj = {
  [`${prefix}-${++i}`]: i, 
  [`${prefix}-${++i}`]: i, 
};

console.log(obj); // {prop-1: 1, prop-2: 2}
```

### 메서드 축약 표현

`function` 키워드를 생략한 축약 표현을 사용할 수 있다.

```js
const obj = {
 name: 'Lee',
 sayName() {
   console.log(`my name is ${this.name}`)
 }
};

obj.sayName() // my name is Lee
```
