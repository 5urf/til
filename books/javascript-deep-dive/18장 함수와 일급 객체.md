## 일급 객체

>[!IMPORTANT]
>**일급 객체**
>
>JS 함수는 일급 객체의 4가지 조건을 모두 충족한다.
>
>일급 객체란 무명 리터럴로 생성가능, 변수/자료구조에 저장 가능, 매개변수로 전달 가능, 반환값으로 사용 가능한 객체를 말한다.

- 함수는 객체이므로 값을 사용할 수 있는 곳이라면 어디서든 리터럴로 정의할 수 있고 런타임에 함수 객체로 평가 된다.
- 함수가 일급 객체라는 것은 함수형 프로그래밍을 가능하게 하는 JS의 장점 중 하나다.

>[!NOTE]
>함수는 객체지만 일반 객체와 달리 호출 가능하고, 일반 객체에는 없는 함수 고유의 프로퍼티(`arguments`, `caller`, `length`, `name`, `prototype`)를 소유한다.

---

## arguments 객체

>[!IMPORTANT]
>**arguments**
>
>`arguments` 객체는 함수 호출 시 전달된 인수 정보를 담는 순회 가능한 유사 배열 객체다.
>
>함수 내부에서 지역 변수처럼 사용되며 외부에서는 참조할  수 없다.

- JS는 매개변수와 인수의 개수 일치 여부를 확인하지 않는다.
- 부족한 인수는 `undefined`로 초기화되고, 초과된 인수는 버려지지 않고 `arguments`객체에 암묵적으로 보관된다.

```js
function sum() {
  let res = 0;
  for (let i = 0; i < arguments.length; i++) {
    res += arguments[i];
  }
  return res;
}

console.log(sum(1, 2));    // 3
console.log(sum(1, 2, 3)); // 6
```

>[!WARNING]
>`arguments`는 유사 배열 객체라 배열 메서드를 직접 사용할 수 없다.
>
>배열 메서드가 필요하면 `Array.prototype.slice.call(arguments)`로 먼저 변환해야 한다.

>[!TIP]
>ES6 Rest 파라미터(`...args`)가 `arguments`를 실질적으로 대체한다.
>
>프로젝트에서도 가변인자 함수는 Rest 파라미터로 구현하는 것이 권장된다.

>[!NOTE]
>`arguments.length`는 인수의 개수, `arguments.callee`는 함수 자신을 가리킨다.
>
>ES6부터 `arguments`는 유사 배열 객체이면서 동시에 이터러블이다.
>
>`Function.arguments`처럼 외부에서 접근하는 방식은 ES3부터 표준에서 폐지되었다.

## length 프로퍼티

>[!IMPORTANT]
>함수 객체의 `length`는 함수 정의 시 선언한 매개변수의 개수를 가리킨다.

>[!WARNING]
>`arguments.length`(실제 전달된 인수 개수)와 `함수.length`(선언된 매개변수 개수)는 다를 수 있다.
>
>혼동하지 않도록 주의해야 한다.

## name 프로퍼티

>[!IMPORTANT]
>`name`프로퍼티는 함수 이름을 나타낸다.
>
>ES5까지는 비표준이었다가 ES6에서 정식 표준이 되었다.

- 익명 함수 표현식의 경우 ES5에서는 빈 문자열을 값으로 갖지만, ES6에서는 함수 객체를 가리키는 변수 이름을 값으로 갖는다.

```js
let anonymousFunc = function() {};
console.log(anonymousFunc.name); // ES6: "anonymousFunc"

function bar() {}
console.log(bar.name); // "bar"
```

## `__proto__` 접근자 프로퍼티

>[!IMPORTANT]
>모든 객체는 `[[Prototype]]`내부 슬롯을 갖는다.
>
>`__proto__`는 이 내부 슬롯이 가리키는 프로토타입 객체에 간접 접근하기 위한 접근자 프로퍼티다.

- `__proto__`는 함수 고유 프로퍼티가 아니라 `Object.prototype`의 접근자 프로퍼티를 상속받은 것이다.
- `[[Prototype]]`내부 슬롯에는 직접 접근할 수 없고 `__proto__`를 통해서만 간접 접근 가능하다.

## prototype 프로퍼티

>[!IMPORTANT]
>`prototype`프로퍼티는 생성자 함수로 호출할 수 있는 함수 객체(constructor)만 소유한다.
>
>일반 객체와 화살표 함수 같은 non-constructor에는 존재하지 않는다.

```js
(function() {}).hasOwnProperty('prototype'); // true
({}).hasOwnProperty('prototype');            // false
```

>[!NOTE]
>`prototype`프로퍼티는 생성자 함수가 생성할 인스턴스의 프로토타입 객체를 가리킨다.
